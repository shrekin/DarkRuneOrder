## v1.0.2 (2026-04-13)

### Bug Fixes
- Fixed taint errors in display.lua — replaced `C_Spell.GetSpellName(spellID)` with `UnitCastingInfo(unitID)` for UNIT_SPELLCAST_START to avoid propagating taint from secret boss spellIDs; STOP/FAILED/INTERRUPTED events now use a local flag instead of touching spellID entirely
- Fixed ADDON_ACTION_FORBIDDEN error on SendChatMessage — was a secondary consequence of the spellName taint contaminating the addon's execution context
- Fixed "Test Cast" button being obscured in test mode — `undoBtn` and `testCastBtn` were both shown at the same anchor point; visibility is now managed per branch in ShowPicker
- Fixed `/dr force` chat announcement going to /say instead of /i — now correctly uses INSTANCE_CHAT when in a raid instance
- Fixed minimap button left-click not recognising raid assistants — now checks `UnitIsGroupAssistant` in addition to `UnitIsGroupLeader`, consistent with the slash command behaviour
- Fixed shared `fadeElapsed` upvalue in `StartFadeIn` — rapid successive calls to `ShowDisplay` no longer corrupt the fade animation; elapsed is now a closure-local variable
- Fixed AuraUtil.FindAuraByName crash on leveling characters — `AuraUtilDataProvider` can be an empty table on low-level chars; now uses `C_UnitAuras.GetAuraDataBySpellName` with a pcall fallback

### New Features
- Raid assistants can now open the picker with `/dr` (previously leader-only)
- Raid assistants now broadcast order via /rw (RAID_WARNING), same as the raid leader
- Non-leader, non-assistant players in a raid now broadcast order via /i (INSTANCE_CHAT) instead of /say
- Pentagon symbol layout is now flipped vertically — point at bottom instead of top
- Close button replaced with custom icon (close-popup-4.png) instead of Blizzard UIPanelCloseButton template
- Custom styled buttons throughout the picker — fill #0E131A, stroke #76787B, Expressway font, no Blizzard templates
- Smooth hover animation on all picker buttons — text and border lerp to #FFFFFF in 0.15s; custom stroke colors auto-brighten by 40%

### Improvements
- SenderIsLeader now also accepts raid assistants (UnitIsGroupAssistant) so ORDER messages from assistants are accepted by all clients
- Button SetText uses rawset to guarantee Lua-level override takes precedence over WoW's native Button:SetText

## v1.0.1 (2025-04-07)

### Bug Fixes
- Fixed SenderIsLeader not working in party groups — previously only checked raid units, now correctly validates leader in both raid and party contexts
- Fixed non-existent font reference (FORCED_NARROWS.TTF) — replaced with bundled Expressway.ttf across picker, roster, and marked modules
- Added validation for incoming symbol IDs — malformed addon messages no longer cause empty display slots
- Fixed variable shadowing in picker.lua — local `positions` no longer shadows the pentagon layout variable

### New Features
- Minimap button — drag to reposition, left-click opens picker, right-click opens test mode. Toggle with `/dr minimap`
- Order history — last 5 symbol orders are saved to SavedVariables, surviving /reload and relog
- Send Last button — quickly re-send the previous order after a wipe or reset
- Undo button — remove the last picked symbol without resetting the entire selection
- Display position persistence — dragged display frame position is saved and restored across sessions
- Raid Warning for leaders — raid leader now broadcasts order via /rw instead of /say for better visibility
- `/dr last` slash command — re-send last order from chat
- `/dr minimap` slash command — show/hide the minimap button

### Improvements
- Sound alert (RAID_WARNING) plays when a new symbol order is received
- Smooth fade-in animation (0.3s) when the display appears
- Incoming ORDER messages are throttled (2s cooldown) to prevent spam
- Graceful degradation for Dark Rune spell name lookup — hardcoded fallback if API returns nil
